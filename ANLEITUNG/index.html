Sinn und Zweck dieses Moduls

Multisite/Multishop erlaubt das Betreiben mehrerer Zen Cart Shops mit einer einzigen Zen Cart Installation.
Artikel, Kunden, usw. werden nur in einer einzigen Administration verwaltet.
Die Shops können unterschiedliche Layouts verwenden und unterschiedlich konfiguriert werden.
Kategorien können in allen Shops oder nur in bestimmten angezeigt werden.
Voraussetzung: Mehrere Domains oder Subdomains, die alle auf dieselbe Zen Cart Installation zielen.

Installation

Voraussetzungen


Dieses Modul ist nur für Zen Cart 1.5.6 deutsch geeignet und ändert zahlreiche sehr wichtige Corefiles.
Es ist kein Modul, das Sie in 2 Minuten einfach hochladen können.

Für den Einbau dieses Moduls sollten Sie über folgende Werkzeuge verfügen:

1) Ein guter Texteditor.
Damit ist nicht das in Windows enthaltene Notepad oder Microsoft Word gemeint.
Sie benötigen einen Texteditor, der utf-8 versteht und auch im Format utf-8 ohne BOM abspeichern kann.
Empfehlung: UltraEdit (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist der kostenlose Texteditor Notepad++

2) Ein Tool zum Vergleichen von Dateien
Bei der Installation dieses Moduls ist es erforderlich, dass Sie den Inhalt einiger Ihrer bestehenden Zen Cart Dateien mit dem Inhalt der neuen Moduldateien vergleichen und die Änderungen zusammenführen ("mergen").
Empfehlung: BeyondCompare (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist das kostenlose Programm WinMerge


Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

Voraussetzung

Sie haben Zen Cart 1.5.6 bereits installiert und einen funktionsfähigen Shop. Diese Anleitung beschreibt, wie Sie Ihren bestehenden Shop modifizieren, um Multisite zu verwenden.

Planung:

Bevor Sie dieses Modul integrieren, legen Sie erst Ihre gewünschte Konfiguration fest. In diesem Beispiel wird von 2 Shops ausgegangen.

1)
Domain
Sie benötigen 2 unterschiedliche Domains, die beide auf dieselbe Zen Cart Installation zielen.
Bsp: www.shop1.de und www.shop2.de
Es könnten auch Subdomains sein, z.B. shop1.meinedomain.de und shop2.meinedomain.de
Oder eine normale Domain und eine Subdomain, z.B. shop1.meindedomain.de und www.meinedomain.de
Richten Sie zunächst in Ihrer Domainverwaltung die beiden Domains so ein, dass beide auf dieselbe Zen Cart Installation zielen

2)
Template
Sie müssen für die beiden Multishops unterschiedliche Templates verwenden.
Bsp:
Shop1 verwendet das Template responsive_classic
Shop2 verwendet das Template classic
Entscheiden Sie sich dafür welcher Shop welches Template verwenden soll.

3)
Jetzt wo Domain und Template feststehen können Sie die Konfigurationsdateien anpassen:

Konfiguration für Shop 1

Öffnen Sie
NEUEDATEIEN/includes/config_sites/www.shop1.de_config.php mit einem Texteditor und ändern Sie folgende Demowerte auf Ihre echten ab:
$template_dir = "responsive_classic";
define('SITE_NAME','shop1');
define('HTTP_SERVER', 'http://shop1.meinedomain.de');
define('HTTPS_SERVER', 'https://shop1.meinedomain.de');
SITE_NAME ist dabei frei wählbar, geben Sie dem Shop einfach einen Namen, der für Sie Sinn macht. Dieser Name wird später intern in der Administration verwendet und sollte eher kurz sein und keine Leer- oder Sonderzeichen enthalten.

Lassen Sie die Werte für Minify auf false, es ist besser Minify, das seit Zen Cart 1.5.3 standardmäßig aktiv ist, für Multishop nicht zu verwenden, da sonst die Stylesheets nicht korrekt geladen werden:

define('MINIFY_STATUS_JS','false');
define('MINIFY_STATUS_CSS','false');

Speichern Sie die Datei nun so ab, dass Ihre echte Domain exakt so enthalten ist, also wenn Ihre echte Domain für Shop 1 www.meinshop1.de lautet, dann speichern Sie als www.meinshop.de_config.php

Konfiguration für Shop 2

Öffnen Sie
NEUEDATEIEN/includes/config_sites/www.shop2.de_config.php mit einem Texteditor und ändern Sie folgende Demowerte auf Ihre echten ab:
$template_dir = "classic";
define('SITE_NAME','shop2');
define('HTTP_SERVER', 'http://shop2.meinedomain.de');
define('HTTPS_SERVER', 'https://shop2.meinedomain.de');
SITE_NAME ist dabei frei wählbar, geben Sie dem Shop einfach einen Namen, der für Sie Sinn macht. Dieser Name wird später intern in der Administration verwendet und sollte eher kurz sein und keine Leer- oder Sonderzeichen enthalten.

Lassen Sie die Werte für Minify auf false, es ist besser Minify, das seit Zen Cart 1.5.3 standardmäßig aktiv ist, für Multishop nicht zu verwenden, da sonst die Stylesheets nicht korrekt geladen werden:

define('MINIFY_STATUS_JS','false');
define('MINIFY_STATUS_CSS','false');

Speichern Sie die Datei nun so ab, dass Ihre echte Domain exakt so enthalten ist, also wenn Ihre echte Domain für Shop 2 www.meinshop2.de lautet, dann speichern Sie als www.meinshop2.de_config.php

Die beiden Beispieldateien www.shop1.de_config.php und www.shop2.de_config.php können Sie nun löschen.

Nach demselben Prinzip könnten Sie natürlich auch noch weitere Shops einbinden.

Nach diesen Vorbereitungen erfolgt die Installation nun in folgenden Schritten. Halten Sie diesen Ablauf ein!

1)
Im Ordner NEUE DATEIEN alle Ordner namens DEINTEMPLATE1 auf das in Ihrem Shop verwendete Template umbenennen, das Sie für Shop 1 verwenden wollen (z.B. responsive_classic). Die Ordner namens DEINTEMPLATE2 auf das in Ihrem Shop verwendete Template umbenennen, das Sie für Shop 2 verwenden wollen (z.B. classic). Den Ordner DEINADMIN auf den Namen Ihres admin Verzeichnisses umbenennen.

3)
Dann alle Dateien/Ordner aus dem Ordner NEUE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen.
Dabei werden keine bestehende Datei überschrieben.

4)
Im Ordner GEAENDERTE DATEIEN alle Ordner namens DEINTEMPLATE1 auf das in Ihrem Shop verwendete Template umbenennen, das Sie für Shop 1 verwenden wollen (z.B. responsive_classic). Die Ordner namens DEINTEMPLATE2 auf das in Ihrem Shop verwendete Template umbenennen, das Sie für Shop 2 verwenden wollen (z.B. classic). Den Ordner DEINADMIN auf den Namen Ihres admin Verzeichnisses umbenennen.
Wenn Sie Zen Cart 1.5.6 gerade frisch installiert haben und noch keinerlei Änderungen an den Dateien vorgenommen haben, können Sie nun alle Dateien/Ordner aus dem Ordner GEAENDERTE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen. Dabei werden dann etliche Dateien überschrieben.
Wenn Sie Zen Cart 1.5.6 schon länger im Einsatz haben und schon einmal Änderungen an Dateien vorgenommen oder andere Module eingebaut haben, dann laden Sie die Dateien keinesfalls einfach hoch.
Vergleichen Sie alle Dateien im Ordner GEAENDERTE DATEIEN mit den entsprechenden Dateien in Ihrem Shop und nehmen Sie die Änderungen manuell per WinMerge oder BeyondCompare vor.
Dann spielen Sie die geänderten Dateien in der gezeigten Struktur ein.
Vermutlich haben Sie in Ihrer Shopinstallation im Ordner includes/init_includes/overrides noch keine Dateien liegen. Vergleichen Sie diese Dateien mit den Originaldateien in Ihrem includes/init_includes Verzeichnis und spilene Sie die geänderten dann nach includes/init_includes/overrides.

5)
Laden Sie folgende Datei aus Ihrer Zen Cart Installation herunter:
includes/configure.php
Fügen Sie direkt am Anfang nach dem öffnenden <?php
Folgendes ein:

include_once('includes/config_sites/sites_switch.php');
Laden Sie die geänderte includes/configure.php wieder hoch und überschreiben Ihre bestehnde damit. Diese Datei ist normalerweise schreibgeschützt (chmod444), daher setzen Sie zunächst die Schreibrechte dieser Datei auf chmod777. Wurde die Änderung erfolgreich übernommen, setzen Sie wieder auf chmod444 zurück.

6)
Testen Sie nun, ob der Aufruf Ihrer beiden Domains korrekt zu den "beiden" Shops führt und ob das jeweils gewünschte Layout angezeigt wird (z.B. Shop 1 erscheint mit template responsive_classic und Shop 2 mit Template classic). Sollte das nicht so sein, dann überprüfen Sie nochmals genau den Inhalt und die Benennung ihrer beiden Konfigurationsdateien in includes/config_sites.

KONFIGURATION

Nachdem nun die grundlegenden Dinge eingestellt sind und funktionieren, können Sie ans Feintuning gehen.

Unter Tools finden Sie in der Zen Cart Administration den neuen Menüpunkt Multisite Tools.

Zeige die Konfigurationsoptionen für eine neue Seite
liefert alle möglichen Konfigurationseinstellungen, die für die beiden Shops unterschiedlich eingestellt werden können.
Wenn Sie in Shop 1 und Shop 2 für iregndwelche dieser Werte unterschiedliche verwenden wollen (und das ist ja eigentlich der Sinn des Multisite Moduls :-)), dann kopieren Sie die gewünschten Einträge jeweils in Ihre beiden Konfigdateien in includes/config_sites

Zeige die Beziehungen zwischen Kategorien und Seiten
Hier können Sie konfigurieren, ob bestimmte Kategorien nur in einem Shop oder auch im anderen angezeigt werden sollen. Somit haben Sie die Möglichkeit, nicht nur Layout und Konfiguration der beiden Shops zu trennen, sondern auch in einem Shop nur bestimmte Kategorien anzuzeigen und im anderen Shop eben nur bestimmte andere.

Optional: Sidebox zum Verlinken des zweiten Shops
Unter Tools > Boxlayout können Sie die Sidebox site_links.php aktivieren.
Die entsprechenden Links und Texte können Sie anpassen in:
includes/modules/sideboxes/DEINTEMPLATE1/site_links.php:
add_site_link('Besuchen Sie auch...','http://shop1.meinedomain.de');
includes/modules/sideboxes/DEINTEMPLATE2/site_links.php:
add_site_link('Besuchen Sie auch...','http://shop2.meinedomain.de');
